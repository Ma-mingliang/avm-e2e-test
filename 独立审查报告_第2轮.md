## 第2轮新发现问题

### 问题 013 - 空仓库初始化的 v1 标签可能与已有标签冲突
- **等级**：一般级
- **证据**：如果用户在已有标签 v1 的仓库上执行 `avm init-project`，工具可能尝试创建重复的 v1 标签。
- **复现场景**：仓库已有 v1.0.0 语义化版本标签，工具误将其识别为正式版本。
- **影响**：版本号冲突。
- **必须修改**：版本号计算必须严格匹配 `^v[0-9]+$`（纯整数），忽略 `v1.0.0` 等语义化版本。初始化时明确列出已识别的正式版本标签，让用户确认。
- **验证方式**：创建包含 v1.0.0 和 v1 标签的仓库，验证版本计算正确。

### 问题 014 - GitHub API 的 refs 端点可能被 GitHub 限制
- **等级**：建议级
- **证据**：GitHub 对 API 调用有速率限制（authenticated 5000/hour）。频繁的锁操作可能触发限流。
- **复现场景**：快速连续执行多个 avm 命令。
- **影响**：锁操作失败。
- **必须修改**：实现重试机制（指数退避），区分限流（429）和锁被占用（422）。
- **验证方式**：mock API 返回 429，验证重试逻辑。

### 问题 015 - 文档本地版本的待归档队列可能被遗忘
- **等级**：建议级
- **证据**：方案说文档本地版本"等待下一正式版本归档"，但如果长期没有正式任务，队列可能积压。
- **复现场景**：用户连续执行10个文档任务，但没有代码任务。
- **影响**：归档延迟。
- **必须修改**：`avm status` 显示待归档文档数量和最早日期。超过30天未归档时发出警告。
- **验证方式**：模拟长期未归档场景，验证警告。
