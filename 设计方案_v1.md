## 3. 状态机设计

### 3.1 合法状态转换矩阵

| 当前状态 | 允许的下一状态 | 触发条件 |
|----------|----------------|----------|
| IDLE | PREFLIGHT | avm preflight |
| PREFLIGHT | WAIT_START_APPROVAL | 预检通过 |
| PREFLIGHT | IDLE | 预检失败 |
| WAIT_START_APPROVAL | RESERVED | 用户批准 |
| WAIT_START_APPROVAL | IDLE | 用户拒绝 |
| RESERVED | LOCKED | 获取锁成功 |
| RESERVED | IDLE | 获取锁失败 |
| LOCKED | BRANCH_READY | 分支创建成功 |
| BRANCH_READY | MODIFYING | 开始修改 |
| MODIFYING | VALIDATING | avm validate |
| MODIFYING | SCOPE_EXPANSION | 范围扩大 |
| VALIDATING | REVIEW_MATERIAL_READY | 验证通过 |
| VALIDATING | FIXING | 验证失败 |
| VALIDATING | DRAFT_PR | 验证失败需草稿PR |
| FIXING | VALIDATING | 修复完成 |
| REVIEW_MATERIAL_READY | WAIT_FINAL_APPROVAL | 材料准备完成 |
| WAIT_FINAL_APPROVAL | PR_READY | 用户批准 |
| WAIT_FINAL_APPROVAL | IDLE | 用户拒绝 |
| PR_READY | MERGING | avm merge |
| MERGING | TAGGING | 合并成功 |
| MERGING | PUBLISH_INCOMPLETE | 合并成功但后续失败 |
| TAGGING | RELEASING | 标签成功 |
| TAGGING | PUBLISH_INCOMPLETE | 标签失败 |
| RELEASING | HANDOFF_UPDATING | Release成功 |
| RELEASING | PUBLISH_INCOMPLETE | Release失败 |
| HANDOFF_UPDATING | CLEANING | 报告更新完成 |
| CLEANING | COMPLETE | 清理完成 |
| COMPLETE | IDLE | 任务结束 |

| 当前状态 | 异常状态 | 触发条件 |
|----------|----------|----------|
| 任意 | INTERRUPTED | 进程中断/用户中断 |

## 5. 安全边界声明

### 5.1 可强制约束

- 通过AVM包装入口运行的Agent操作
- 正常Git/GitHub权限模型下的操作
- 本地Git Hooks拦截（可被--no-verify绕过）
- 远程Ruleset和CI检查（需GitHub计划支持）

### 5.2 无法强制约束

- 拥有管理员权限可关闭Ruleset
- 高权限Token可绕开工具直接操作
- Agent可直接修改本地文件（AVM只能在提交和发布时阻止）
- 使用--no-verify可绕过本地Hooks

### 5.3 准确表述

> AVM对通过其包装入口和正常Git/GitHub权限模型运行的Agent提供强制、可审计的多层版本工作流；远程Ruleset和CI负责阻止大部分本地绕过，但不能对仓库管理员或泄露的高权限凭据提供绝对防护。

## 6. 异常场景推演

### 6.1 新空仓库初始化

1. 用户批准初始化
2. 创建私有仓库
3. 创建受控初始提交并推送默认分支
4. 立即安装Ruleset、CI和Hooks
5. 创建v1标签和Release
6. 在v1报告中明确记录该引导例外

### 6.2 PR合并后Release失败

1. 状态转为PUBLISH_INCOMPLETE
2. 原版本号继续占用
3. 禁止新任务
4. 不删除任务分支和锁
5. 重试时确认主分支提交未变化
6. 成功后恢复正常闭环

### 6.3 审批后文件变化

1. 计算approval_hash = SHA256(base_commit + sorted(file_path + file_sha256) + ...)
2. 审批后重新计算
3. 不一致则状态转为APPROVAL_INVALIDATED
4. 必须重新审批

### 6.4 网络中断

1. 本地工作保留
2. 禁止完成发布
3. 保留任务分支、锁和版本号
4. 网络恢复后继续

## 7. Windows特殊处理

### 7.1 路径处理

- 使用pathlib处理路径
- 支持长路径（\\?\前缀）
- 中文路径使用UTF-8编码
- 空格路径使用引号保护

### 7.2 PowerShell编码

- 脚本使用UTF-8 BOM
- 输出使用chcp 65001
- 参数传递使用数组而非字符串拼接

### 7.3 文件锁

- 使用filelock库
- Windows使用msvcrt.locking
- 超时后自动释放
